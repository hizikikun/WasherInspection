# WSL2とは？

## 📖 基本的な説明

**WSL2**（Windows Subsystem for Linux 2）は、**Windows上でLinuxを動かすためのシステム**です。

### 簡単に言うと

- Windowsのパソコンで、Linux（Ubuntuなど）を使えるようにする機能
- 仮想マシンのようにLinux環境を動かせる
- **WindowsとLinuxを同時に使える**

## 🎯 なぜWSL2を使うのか？

### このアプリケーションの場合

1. **Windowsネイティブ環境**:
   - TensorFlowがCPU版しか使えない
   - GPUを直接認識できない

2. **WSL2環境**:
   - ✅ TensorFlowがGPUを完全に認識できる
   - ✅ RTX 4070をフル活用できる
   - ✅ 学習速度が10-50倍速くなる

## 💻 実際の使い方

### WSL2を使う前（Windowsネイティブ）

```
Windows環境
├─ Python（CPU版TensorFlow）
└─ GPU検出: ❌ 認識できない
```

### WSL2を使う（推奨）

```
Windows環境
└─ WSL2（Linux環境）
    ├─ Python（GPU版TensorFlow）
    └─ GPU検出: ✅ 完全認識（RTX 4070使用可能）
```

## 🔧 WSL2のセットアップ

### 1. WSL2は既にインストール済み

あなたのPCには既にWSL2がインストールされています（Ubuntu-22.04）。

### 2. TensorFlow GPU環境もセットアップ済み

`setup_wsl2_tensorflow_gpu.sh`を実行して、GPU対応のTensorFlowをインストール済みです。

### 3. 使うだけ

アプリケーションから「WSL2 GPUモード」を選択するだけで、自動的にWSL2環境で学習が実行されます。

## 🤔 よくある質問

### Q1: WSL2を使うとWindowsアプリが動かなくなる？

**A:** なりません。WindowsとWSL2は同時に使えます。

### Q2: ファイルは共有できる？

**A:** はい。Windowsのファイル（`C:\Users\tomoh\WasherInspection`）は、WSL2から`/mnt/c/Users/tomoh/WasherInspection`としてアクセスできます。

### Q3: 学習中に他のアプリも使える？

**A:** はい。WindowsアプリもWSL2も同時に動かせます。GPU学習中でも、ブラウザや他のアプリは普通に使えます。

### Q4: WSL2は常に起動している必要がある？

**A:** いいえ。必要な時だけ自動的に起動されます。このアプリケーションがWSL2を使って学習する時だけ起動します。

### Q5: パスワードや設定は必要？

**A:** このアプリケーションから使う場合は、すべて自動で処理されます。手動でWSL2を使う場合のみ、初回起動時にパスワード設定が必要です。

## 🚀 まとめ

- **WSL2 = Windows上でLinuxを使う機能**
- **GPU学習にはWSL2が最適**（WindowsネイティブではGPUが使えない）
- **既にセットアップ済み**なので、選択するだけ
- **高速学習が可能**（RTX 4070をフル活用）

**結論**: WSL2を選ぶと、GPUを使って超高速で学習できます！🎯
